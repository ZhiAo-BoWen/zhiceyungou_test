# 智策云构 Pro — 项目需求说明

> 产品名称：智策云构 Pro · 版本 1.0.0  
> 定位：基于业务场景的软硬件选型与项目结构生成引擎（Agent 方向）

---

## 一、技术选型

| 类别 | 方案 | 说明 |
|------|------|------|
| 应用形态 | 本地 Web 应用 | 突破浏览器沙盒，支持本地文件夹选择与写入 |
| 后端框架 | Python + Flask | 轻量快速，便于本地部署 |
| Agent 框架 | LangChain | 驱动大模型完成解析与分析 |
| 大模型 | 通义千问 DashScope | OpenAI 兼容 API，默认 `qwen3.7-plus` |
| 前端 | HTML / CSS / JavaScript | Mermaid 图表、Chart.js 可视化、Lucide 图标 |
| 运行环境 | Conda | 环境名：`agent`，Python 3.11+ |

---

## 二、核心功能

### 2.1 业务输入

- **表单输入**：填写项目名称、业务领域、业务描述、用户规模等字段（必填项标红星）
- **文件上传**：支持 txt / md / pdf / doc 等格式，AI 自动解析为结构化表单
- **大模型选择**：可在多个有免费额度的模型间切换
- **清除按钮**：表单与文件上传均支持一键重置

### 2.2 智能分析

1. 判断输入类型：表单直接使用，文本文件先解析为表单
2. 基于表单内容，调用大模型生成：
   - 技术栈选型表 + 技术架构 Mermaid 图
   - 硬件推荐方案 + 多维度雷达图可视化
   - 项目目录结构树 + 结构关系 Mermaid 图
3. 任务状态：进行中 → 已完成 / 已失败，支持历史回看

### 2.3 结果展示

分析完成后，通过右上角三个 Tab 切换模块（不同时展示）：

| 模块 | 内容 |
|------|------|
| 技术选型 | 技术栈表格 + 技术架构图（支持全屏缩放 / 下载 PNG） |
| 硬件推荐 | 硬件选型表 + 各硬件性能雷达图 |
| 项目结构 | 左侧方案说明与结构图；右侧本地资源管理器 |

### 2.4 工作空间

- 用户可选择本地文件夹作为工作空间，路径绑定到任务记录
- 资源管理器实时同步目录，支持折叠/展开、多选、新建/删除/重命名
- 一键将 AI 生成的项目结构写入工作空间

## 三、UI 设计

### 3.1 布局

- **左侧导航栏**（固定高度）：新建任务、任务历史、系统介绍
- **最近任务列表**：每条记录可点击，导航高亮与右侧内容保持一致
- **右侧主区域**：根据导航切换视图

### 3.2 侧边栏底部

- 第一行：健康检查、API Key 配置
- 第二行：深浅色主题切换

### 3.3 其他交互

- 分析等待页：加载动画 + 项目名称 + 所选模型 + 等待提示
- 图表全屏：滚轮缩放、拖拽平移、重置视图、导出 PNG
- API Key 通过弹窗配置，写入本地 `.env` 并立即生效

## 五、配置与安全

### 5.1 环境变量

复制 `.env.example` 为 `.env`，填写以下配置：

```env
DASHSCOPE_API_KEY=your-api-key-here
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DEFAULT_MODEL=qwen3.7-plus
FLASK_SECRET_KEY=change-me
```

> **注意**：API Key 仅保存在本地 `.env` 文件中，不得提交到 Git 仓库。

### 5.2 启动方式

```bash
conda activate agent
pip install -r requirements.txt
python app.py
```

浏览器访问：http://127.0.0.1:5000

## 六、目录结构

```

├── app.py                 # Flask 入口
├── config.py              # 应用配置（品牌、模型、版本等）
├── agents/analyzer.py     # LangChain 解析与分析 Agent
├── services/              # 任务存储、文件解析、工作空间、健康检查等
├── templates/             # 页面模板
├── static/                # CSS / JS / 图标 / 头像
├── data/                  # 任务历史与上传文件（本地运行时生成）
├── examples/              # 业务需求示例
├── .env.example           # 环境变量模板
└── requirements.txt       # Python 依赖
```
